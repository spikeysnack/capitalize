NAME

capitalize
Capitalize or change case of a list of filenames.

USAGE

capitalize  [options]  [file | file list]
capitalize  -h

OPTIONS

                    -f         --force         rename files without question (dangerous)
                    -h         --help          print this message
                    -i         --interactive   ask for each file before renaming
                    -l         --lower         uncapitalize all letters in filenames
                    -r         --recursive     recursively change all filenames in subdirectories
                    -t         --test          test mode -- do not actually change filenames 
                    -T         --Title         Capitalize With Titlecase
                    -u         --upper         CAPITALIZE ALL LETTERS IN FILENAMES
                    -v         --verbose       print out lots of things as they happen 
           (hint:  use * for all files in dir)

DESCRIPTION

capitalize changes capitalization of filenames in bulk.

Its use is straightforward compared to other methods.

EXPLANATION

Changing the case or capitalization of a group of files and/or directories

is a chore, and there are few straightforward ways to do it.

Capitalize makes it quick and easy. 


The most common use for capitalize 
would be for changing the capitalization of
a group of files in a directory or directory hierarchy.

  

INTERACTIVE

Interactive mode is the safest if not the most efficient way to use capitalize. 


It asks the user for a yes or no before each file is renamed.



non-interactice mode (default)is quite convenient and fast, 
but should not be used thoughtlessly,
or serious messing-up of directories full of files may result.

Test mode is the recommended way to see what would happen. Please.



BUGS

The program will abort if changing the case of a file would overwrite another,
separate file with the target capitalization in the same directory.
This could leave a directory hierarchy of files incompletely changed.
